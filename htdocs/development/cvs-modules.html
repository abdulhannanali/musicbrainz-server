This page describes the various modules you will find in our CVS repository.

About CVS Tags

In all cases by doing a plain "checkout" of the appropriate module you'll be getting the latest development copy, which won't necessarily be stable.  If you want a stable copy, corresponding to a particular release, you'll need to find the correct CVS tag corresponding to the release you want; you can then fetch the release in question.

For example, to fetch the 2.0.2 release of mb_client you'd use a command like this: cvs checkout -r release-2-0-2 mb_client .  Whereas to fetch the latest, unstable version you'd omit the "-r" option: cvs checkout mb_client .

Here is the list of CVS modules available; each module has its own set of release numbers / tags which you can use if desired.

Current Modules
mb_server MusicBrainz Server source code -- this contains all the sources necessary to set up a MusicBrainz server.  Most of this code is in Perl and is designed to run inside the Apache web server in mod_perl with the Mason web framework.
mb_client MusicBrainz Client source code -- this module contains the sources for building the libmusicbrainz library, which provides low level access to the MusicBrainz server and the ability to look up TRM ids from WAV audio data.
tunepimp TunePimp library source code -- this module contains the sources for building the libtunepimp library.  This library encapsulates the core tagging functionality of the MusicBrainz Tagger.  Using this library, it's easy to write a complete tagging application -- only a UI layer needs to be added.  This library handles all the tricky details of reading and writing metadata tags, looking up TRM ids from various audio formats, and matching local audio files with tracks in the MusicBrainz server.  Also includes Perl bindings.
mb_tagger MusicBrainz Tagger Application source code -- this module contains all the sources necessary to build the MusicBrainz tagger application.  Borland C++ builder 6.0 is required to build this module.
python-musicbrainz Python bindings for libmusicbrainz -- these bindings allow Python developers to use the MusicBrainz client library.
irc-logs Archived chat logs -- this module contains the archives of chat logs from the #musicbrainz channel on IRC chat at irc.freenode.net.  The logs are saved to CVS once a day, so CVS usually doesn't contain the last 24 hours' data.  You can browse all the logs, and see the most up-to-date logs, at our IRC log archives site.

Deprecated Modules
cdi_client CD Index client source code -- this module contains the old CD Index client library.  Superseded by the mb_client project.
cdi_lib CD Index client library code -- contains code for the CD Index client library.  Superseded by the mb_client project.
cdi_server CD Index server code -- contains all the code for the old CD Index server.  Superseded by mb_server.
charity An old text-based tagging application.  Superseded by the example program "tp_tagger" in the tunepimp project.
old.tunepimp This module contains the first attempt at a tunepimp library.  Superceded by the tunepimp project.
trm This small library originally looked up TRM ids, but this functionality has now been wrapped into the tunepimp library.
trm_plugin This is the TRM plugin for Bitzi's Bitcollider.

The latest bleeding edge version of our client library and server source is always available to everyone interested by read-only CVS. Currently the CVS archives are still named for their predecessor counterparts in the CD Index. In the next few days these will be moved over to the MusicBrainz repository.

Changes to the source trees are commented, releases are labeled. So you can find out what was going on and retrieve older releases if necessary.

This system is well suited to administer changes to a project made by a group of various developers and is thus used by many large Internet projects.


Further reading

For more information visit the Cyclic Software site or have a look at the info documentation that is online here at the FreeBSD Project or here at Apple Computer, Inc.


Preparation

Before you can access the CVS server you need to set up an environment variable.

Under UNIX using a Bourne shell (sh, bash etc) type the following:

CVSROOT=:pserver:cdindex@cvs.freeamp.org:/src/repository
export CVSROOT
or
setenv CVSROOT :pserver:cdindex@cvs.freeamp.org:/src/repository

if you work under a C shell (csh, tcsh etc).

Under Windows type the following:

set CVSROOT=:pserver:cdindex@cvs.freeamp.org:/src/repository

Alternately you can specify the CVSROOT when you give a CVS command by using the -d flag. While this allows you to access multiple CVS servers at a time, it does require you to specify the server each time you login to or checkout from the repository.

cvs -d:pserver:cdindex@cvs.freeamp.org:/src/repository command


Login

In order to login to the server type:

cvs login

And when prompted use the password:

cdindex1

Under UNIX, after the first time you log into the CVS server the password should be saved in ~/.cvspass.


Getting the source trees

Now all you have to do to download the server source code is type:

cvs -z3 checkout -P cdi_server

To download the client source code do:

cvs -z3 checkout -P cdi_client

and to download the client library source code enter this:

cvs -z3 checkout -P cdi_lib

Note that he -z3 option will use a moderate level of compression on the data exchanged with the server.


Updating

Once you have checked out a copy of the source you can retrieve the latest version by changing to the project directory and typing:

cvs -z3 update -Pd


Retrieving old releases

Everytime we have the source ready for a release, we tag the files at that version with an adequate label. To find out what labels exist, run the following command:

cvs -z3 status -v

Retrieving the version that was in the 0.4.0 client release is as easy as

cvs -z3 update -rCDI_0_4_0 ChangeLog

To restore it back to the most recent version enter:

cvs -z3 update -A ChangeLog

Note that he -A option is necessary to pull you back on the main trunk of the development source tree.

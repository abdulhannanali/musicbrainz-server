<& /comp/sidebar, title=>'Automatic Insertion Stats', expand=>'information' &> <%perl> my $mb = MusicBrainz->new(); $mb->Login(); my ($pe, $obj, @ref, $bgcolor); $pe = Pending->new($mb->{DBH}); @ref = $pe->LoadInsertHistory(50); These tracks where collected from FreeAmp users who signatured their MP3 collection and contributed the metadata from their MP3/Vorbis files to MusicBrainz. A track with the same TRM Id will only get accepted into the system if MusicBrainz has seen two different files with the same ID3 tags. Why is this? Simple -- if two different files with the same TRM Id have the same metadata, it suggests that the metadata for the files is likely to be correct. This cuts down on the amount of garbage metadata that gets inserted into MusicBrainz.

So, if you just signatured your files with FreeAmp, and you wonder why you're files are not showing up here, its probably because MusicBrainz has not seen the files you've submitted twice, thus not triggering the data insertion process. If you have a TRM Id you'd like to look up and see if it is in the Pending tables (waiting to have a second file found), please use the TRM Id lookup feature on the search page.

%# vi: set ts=2 sw=2 ft=mason : <& /comp/sidebar, title => 'November 4th, 2004: TRM Database Pruned' &>

At about 1930hrs UTC on August 9th the TRM database was "pruned" again (see the previous time for more information about this).  This time we removed all TRMs apart from the ones attached to MusicBrainz tracks, where the TRM had been looked up at least once.  This is a slightly more aggressive prune compared to the previous time. 

The TRM database is now about 40% of the size it was before, which (like last time) means that it now fits easily into the server's memory, so the server as a whole runs quickly and reliably.

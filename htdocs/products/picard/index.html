PLEASE READ THIS ENTIRE PAGE BEFORE REPORTING BUGS/PROBLEMS!

How to get the new tagger

To download this new (beta) project, go to the Helix Community download page.

Things that do work:

  1. Open files into [Drags tracks here] by doubleclicking file
  2. Drag files, and directories into [ drag tacks here ]
  3. Drag files into specific album slots
  4. Drag files/dirs into album name slot, auto match tracks to album
  5. Save files
  6. Remove files/albums
  7. Cluster unmatched files into album clusters
  8. Basic preference dialog features

Things that need to be done, but are not done:

General

  • Not all options are hooked up yet.
  • No keyboard shortcuts for drag and drop operations
  • There is no way to generate a TRM for a file
  • All files that have no MB ids in their tags will always end up in the unmatched tracks folder when dropped onto [Drag your tracks here] -- use the Cluster button to cluster these unmatched files into album clusters.
  • The application exits without asking the user to save unsaved files
  • The genre and comment fields are not read/written from/to music files
  • Add Helix Community support by using PyPlayer for music preview and for mp3 decode support.

    Album Panel

  • Should be able to drag albums onto other albums to merge them
  • Multiple select in the album view does not work

    File Browser

  • The file browser does not get updated when files/dirs are saved
  • The file details should show up in Local mdata when file clicked
  • Files should have appropriate icons
  • Dragging directories out of the directory browser will most of the time not fully release the dragged item from the dir browser. The drag and drop operation will still work, but the dragged item is still 'stuck' to the mouse cursor when the mouse cursor is moved over the dir browser. Double click in the white area of the dir browser to 'unstick' this item. This happens under windows only.
  • Dragging multiple files from the file listing will cause the all but the clicked item to deselect just before dragging starts. The application is smart enough to still drag the right number of items, but it doesn't look right -- this happens under Linux only.

    Metadata Information Panel

  • The artist and album combo box drop downs do not populate with correct data

    Toolbar, Menus and Appearance

  • The application needs a toolbar
  • The edit menu cut, copy and paste options are not functional yet.
  • No hourglass cursors are in use yet.

    Platform Specific Issues

  • The application has not been tuned for Mac OS X
  • Application currently runs as a UNICODE application, but ID3 tags are still written in ISO-8859-1

    Reporting Bug and Project Mailing Lists

    Please report any bugs you find here. Enhancement requests may be file here.

    If you'd like to participate in the discussion about this project, please join the Helix community's MusicBrainz-dev mailing list.
